## 1. Acceptance Criteria  
1. **Retry Count** – A failed tool call is retried up to **3 times** (original attempt + 2 retries).  
2. **Back‑off Timing** – The wait between retries follows an exponential sequence: **500 ms → 1 s → 2 s**.  
3. **Success Rate** – After retries, the overall success rate for tool calls must be **≥ 99.9 %** under normal load.  
4. **Root‑Cause Logging** – Each retry attempt logs an error entry that includes a **categorical label** (`RATE_LIMIT`, `NETWORK`, `UNKNOWN`).  
5. **Log Format** – Log entries contain:  
   - Timestamp (ISO‑8601)  
   - Tool name  
   - Attempt number  
   - Error message  
   - Category label  
6. **No Duplicate Logs** – For a single failure, exactly one log entry per retry attempt is emitted.  
7. **Graceful Degradation** – If the error cannot be classified, the category defaults to `UNKNOWN` and the system continues retrying until the cap.
