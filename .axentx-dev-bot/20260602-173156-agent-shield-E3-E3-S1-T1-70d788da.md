import psutil
import logging
import time
from config import MEMORY_LIMIT, MEMORY_THRESHOLD, ALERT_CHANNEL

logger = logging.getLogger(__name__)

def get_memory_usage():
    process = psutil.Process()
    return process.memory_info().rss / (1024 * 1024)  # in MB

def get_top_memory_functions():
    import line_profiler
    profiler = line_profiler.LineProfiler()
    profiler.add_function(get_memory_usage)
    profiler.run("get_memory_usage()")
    profiler.print_stats()
    return profiler.get_line_stats()[:5]

def run_profiler():
    memory_usage = get_memory_usage()
    if memory_usage > MEMORY_THRESHOLD * psutil.virtual_memory().total / (1024 * 1024):
        logger.warning(f"Memory usage exceeded {MEMORY_THRESHOLD * 100}% threshold")
        # Send alert to Ops channel
        send_alert(ALERT_CHANNEL, f"Memory usage exceeded {MEMORY_THRESHOLD * 100}% threshold")
    top_functions = get_top_memory_functions()
    logger.info(f"Top 5 memory-heavy functions: {top_functions}")
    # Log top 5 memory-heavy functions
    log_top_functions(top_functions)

def schedule_profiler():
    while True:
        run_profiler()
        time.sleep(600)  # 10 minutes

def main():
    schedule_profiler()

if __name__ == "__main__":
    main()

## 1. Acceptance Criteria
1. **Periodic execution** – The profiler runs automatically every 10 minutes (±5 seconds) without external trigger.  
2. **Peak RSS capture** – For each run it records the *peak* resident‑set‑size (RSS) of the agent process and writes the value (in MB) to `memory_profiler.log` with a timestamp.  
3. **Threshold alert** – If the recorded peak exceeds **80 %** of the `AGENT_MEMORY_LIMIT_MB` environment variable, an alert is sent to the configured Ops channel (Slack webhook URL in `OPS_ALERT_WEBHOOK`).  
4. **Top‑5 memory‑heavy call sites** – The profiler logs the five functions (module + line) that contributed the most to RSS, ordered by descending memory consumption, in the same log entry.  
5. **Non‑blocking** – Profiling runs in a background thread/process; it must not block the main agent loop. CPU usage of the profiler thread stays ≤ 2 % on an idle agent (measured over a 30‑second window).  
6. **Graceful degradation** – If `psutil` cannot be imported, or the OS does not expose memory maps, the profiler disables itself, writes a warning to the log, and never raises an exception.  
7. **Observability** – Each profiling run emits a structured log entry (JSON) containing: `timestamp`, `peak_mb`, `threshold_exceeded` (bool), `top_functions` (list of `{module, line, rss_mb}`).
