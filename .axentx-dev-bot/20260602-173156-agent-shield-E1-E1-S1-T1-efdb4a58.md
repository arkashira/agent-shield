## 1. Overview  
TCG‑Analytics continuously consumes code‑coverage reports produced by existing test runners (e.g., **JaCoCo**, **coverage.py**, **nyc**, **cobertura**) and applies lightweight statistical models to surface *coverage gaps* and *trend anomalies*. It surfaces actionable recommendations through:

* a **RESTful API** (JSON) for downstream tools,  
* a **GitHub/GitLab Pull‑Request comment bot**, and  
* a **CLI** for ad‑hoc queries.

The service is packaged as a **Docker‑based microservice** that can be deployed on‑prem or in any Kubernetes cluster and integrates with any CI system via a single step that uploads the latest coverage artefact.

---

## 2. Architecture Diagram  

```
+-------------------+        +-------------------+        +-------------------+
|   CI/CD Runner    |  -->   |   Coverage Upload |  -->   |   TCG‑Analytics   |

| Actions, GitLab)  |        +-------------------+        +-------------------+
+-------------------+                                      |
                                                          |
                +-------------------+   +-----------------+-----------------+
                |   ML/Stats Engine |   |   Recommendation Engine           |
                +-------------------+   +-----------------------------------+
                              |                     |
                +-------------+-------------+-------+-------------------+
                |                           |                       |
          +-----------+               +-----------+           +-----------+
          |  API GW   |               |  Bot (PR) |           |  CLI Tool |
          +-----------+               +-----------+           +-----------+
```

* **Coverage Upload** – a tiny HTTP endpoint (`/upload`) that receives a coverage report (XML/JSON) and a git SHA.  
* **ML/Stats Engine** – computes per‑file, per‑function, and per‑branch coverage statistics; runs a **Isolation Forest** to flag outliers (e.g., sudden drops).  
* **Recommendation Engine** – maps gaps to *template suggestions* (e.g., “Add unit test for `processOrder()` handling null input”).  
* **API GW** – FastAPI (Python) exposing `/coverage`, `/recommendations`, `/trend`.  
* **Bot** – GitHub Action / GitLab CI job that posts a comment on the PR with a markdown table of the top‑5 gaps.  
* **CLI** – `tcg-cli` (installed via pip) for local inspection (`tcg-cli trend --since 30d`).
