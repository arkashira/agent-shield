```python
import logging
from logging.handlers import RotatingFileHandler
from datetime import datetime, timedelta
import json
import os

# Define the audit log file path and rotation settings
AUDIT_LOG_FILE = '/opt/axentx/agent-shield/logs/audit.log'
MAX_LOG_SIZE = 10 * 1024 * 1024  # 10MB
BACKUP_COUNT = 365  # 1 year

# Create a rotating file handler for immutable audit logs
handler = RotatingFileHandler(AUDIT_LOG_FILE, maxBytes=MAX_LOG_SIZE, backupCount=BACKUP_COUNT)
handler.setLevel(logging.INFO)

# Define a custom logging format
formatter = logging.Formatter('%(asctime)s - %(name)s - %(levelname)s - %(message)s')
handler.setFormatter(formatter)

# Create a logger instance
logger = logging.getLogger('audit_logger')
logger.setLevel(logging.INFO)
logger.addHandler(handler)

# Function to log tool calls
def log_tool_call(user_id, timestamp, operation_details):
    log_data = {
        'user_id': user_id,
        'timestamp': timestamp,
        'operation_details': operation_details
    }
    logger.info(json.dumps(log_data))

# Function to query audit logs by date range and user ID
def query_audit_logs(start_date, end_date, user_id):
    logs = []
    with open(AUDIT_LOG_FILE, 'r') as file:
        for line in file:
            log_data = json.loads(line.strip())
            if start_date <= log_data['timestamp'] <= end_date and log_data['user_id'] == user_id:
                logs.append(log_data)
    return logs
```

## 1. Acceptance Criteria
1. **Tool‑call logging** – Every invocation of a registered tool creates a single audit record containing:
   - `user_id` (string, non‑empty)  
   - `timestamp` (ISO‑8601 UTC, ≤ 1 ms drift from `datetime.utcnow()`)  
   - `tool_name` (string, matches a name in `ToolRegistry`)  
   - `operation` (string, e.g., “execute”, “configure”)  
   - `details` (JSON‑serialisable dict, may be empty)  

2. **Memory‑operation logging** – All reads and writes performed by a tool are logged with:
   - `user_id`  
   - `timestamp`  
   - `memory_key` (string)  
   - `action` (`"read"` or `"write"`)  
   - `value_snapshot` (for writes: JSON‑serialisable copy of the value; for reads: `null`)  

3. **Immutability & retention** – Audit logs are written to a directory that:
   - Uses a `RotatingFileHandler` (or `TimedRotatingFileHandler`) with **daily rotation** and **backupCount ≥ 365**.  
   - Opens files in **append‑only** mode (`mode='a'`) and never rewrites existing entries.  
   - Sets file permissions to read‑only for non‑owner processes (e.g., `chmod 0440`).  

4. **Query endpoint** – `query_audit_logs(start_date, end_date, user_id=None)` returns a **list of dicts** ordered chronologically, supporting:
   - Inclusive date‑range filter (`start_date` ≤ entry.timestamp ≤ `end_date`).  
   - Optional exact `user_id` filter.  

5. **Performance** – For a log file containing **1 M entries**, a query covering a 30‑day window must complete **≤ 500 ms** on the reference CI runner (2 vCPU, 4 GB RAM).  

6. **Tamper resistance** – Any attempt to modify or delete an existing log entry raises a custom `AuditLogTamperError` and leaves the underlying file unchanged.  

7. **Robustness to corruption** – Malformed JSON lines are **skipped**, a warning is emitted via the standard logger, and the query continues without raising an exception.
