## 1. Acceptance Criteria  

| # | Criterion | Measurable Target |
|---|-----------|-------------------|
| 1 | **Default memory cap** | Each agent’s resident set size (RSS) never exceeds **512 MiB** unless the limit is overridden. |
| 2 | **Configurable limit** | The environment variable `AGENT_MEMORY_LIMIT` (value in MiB) overrides the default; the effective limit equals `value * 1024 * 1024` bytes. |
| 3 | **Warning threshold** | When RSS > 80 % of the configured limit, AgentOS writes a **WARN** log entry containing `agent_id`, `memory_usage_bytes`, and `threshold="80%"`. |
| 4 | **Leak detection** | If RSS stays **≥ 90 %** of the limit for a continuous **5 minutes**, an **ERROR** log entry is emitted with `agent_id`, `memory_usage_bytes`, `duration_seconds`, and a captured stack trace. |
| 5 | **Prometheus metric** | A gauge metric `agent_memory_usage_bytes{agent_id="<id>"}` is exported every **10 seconds** and reflects the current RSS in bytes. |
| 6 | **Graceful termination** | When RSS > limit, the runtime sends **SIGTERM** to the agent process; the process must exit with status 0 after flushing pending logs (timeout ≤ 30 s). |
| 7 | **Observability format** | All memory‑related logs are emitted via the central JSON logger and are searchable by the tag `memory_event`. |
