## 1. Acceptance Criteria
| # | Criterion | Measurable Target |
|---|-----------|-------------------|
| 1 | **Default memory cap** | Each agent’s resident memory (RSS) must never exceed **512 MiB** unless overridden by the `AGENT_MEMORY_LIMIT` environment variable. |
| 2 | **Configurable limit** | Setting `AGENT_MEMORY_LIMIT` to an integer **N** (MiB) changes the per‑agent cap to **N MiB** (validated at agent start‑up). |
| 3 | **Warning log** | When an agent’s RSS > **80 %** of its configured limit, a log entry with level **WARN**, message containing `memory_warning`, and the agent ID is emitted **once per breach**. |
| 4 | **Prometheus metric** | A gauge metric `agent_memory_usage_bytes{agent_id="<id>"}` is exported every **5 seconds** and reflects the current RSS in bytes. |
| 5 | **Leak detection** | If RSS stays > **90 %** of the limit for a continuous **5 minutes**, an **ERROR** log entry `memory_leak` is emitted with a stack trace of the allocation hotspot. |
| 6 | **Graceful termination** | When RSS exceeds the absolute limit (100 %), the agent receives **SIGTERM**, flushes logs, and exits with status 0 within **30 seconds**. |
| 7 | **Observability** | All logs and metrics are tagged with the agent’s unique identifier (`agent_id`). |
