## 1. Acceptance Criteria  
*(Each criterion must be objectively measurable by automated tests or observable metrics.)*

1. **Default memory cap** – An agent process must not exceed **512 MiB** of resident memory unless the environment variable `AGENT_MEMORY_LIMIT` overrides it.  
2. **Configurable limit** – If `AGENT_MEMORY_LIMIT` is set (value in MiB), the cap equals that value; invalid or missing values fall back to the default 512 MiB.  
3. **Warning threshold** – When an agent’s RSS exceeds **80 %** of its configured limit, AgentOS writes a log entry at level **WARN** containing the fields `event="memory_warning"` and `agent_id`.  
4. **Leak detection** – If an agent’s RSS stays above **90 %** of its limit for a continuous **5 minutes**, AgentOS writes a log entry at level **ERROR** with `event="memory_leak"`, `agent_id`, and a captured stack trace.  
5. **Prometheus metric** – A gauge metric `agent_memory_usage_bytes{agent_id="<id>"}` is exposed and updated at least once per second with the current RSS.  
6. **Graceful termination** – When an agent’s RSS exceeds **100 %** of its limit, AgentOS sends a `SIGTERM` to the agent process, waits up to 30 seconds for a clean shutdown, then forces `SIGKILL` if the process is still alive.  
7. **Observability** – All log entries must be emitted to the standard AgentOS logger and be searchable via the existing log aggregation pipeline.
