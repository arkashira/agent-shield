import time
import random

def call_with_retry(func, max_retries=3, initial_backoff=0.5):
    """
    Calls the given function with retry mechanism.

    Args:
    - func: The function to be called.
    - max_retries: The maximum number of retries.
    - initial_backoff: The initial back-off time in seconds.

    Returns:
    - The result of the function call if successful, otherwise None.
    """
    backoff = initial_backoff
    for attempt in range(max_retries + 1):
        try:
            # Attempt to call the function
            result = func()
            # If successful, return the result
            return result
        except Exception as e:
            # If not successful, check if we've reached the max retries
            if attempt < max_retries:
                # Calculate the back-off time with jitter
                backoff_with_jitter = backoff * (1 + random.uniform(0, 0.1))
                # Sleep for the back-off time
                time.sleep(backoff_with_jitter)
                # Exponentially increase the back-off time
                backoff *= 2
            else:
                # If all retries fail, raise the last exception
                raise e
    # If we've reached this point, all retries have failed
    return None

## 1. Acceptance Criteria
1. **Transient‑error retry** – Any tool call that returns an HTTP 5xx status code or times‑out is automatically retried up to **`MAX_RETRIES`** times (default = 3).  
2. **Exponential back‑off** – Delay before retry *i* = `BASE_DELAY_MS * 2^(i‑1)` (BASE_DELAY_MS = 100 ms), capped at **5 seconds**.  
3. **Configurable max retries** – The maximum number of retries is read from the environment variable `MAX_RETRIES`. Missing, empty, or non‑numeric values fall back to the default (3).  
4. **Logging** – Each retry attempt logs: tool identifier, attempt number, error type (HTTP status or timeout), and calculated back‑off duration. After the final failure, a summary log includes total attempts and final error.  
5. **Circuit breaker – open** – For a given tool, after **5 consecutive** failed calls (i.e., after all retries are exhausted) the circuit breaker opens, rejecting further calls instantly with a `CircuitBreakerOpen` error.  
6. **Circuit breaker – cooldown** – While open, the breaker blocks calls for **60 seconds**. After the cooldown the breaker automatically resets to the closed state.  
7. **Metrics exposure** – The module exposes two counters (via a Prometheus‑compatible interface): `tool_retry_total{tool}` and `circuit_breaker_open_total{tool}`.  

All criteria are observable via unit/integration tests, log inspection, and metric queries.
