**Unified, production‑ready solution for a resilient “retry + circuit‑breaker” layer**

Below is a single, coherent implementation that merges the strongest points from the three draft proposals, resolves their contradictions, and gives you a **concrete, testable** artifact you can drop into the `agent‑shield` code‑base today.

---

## 1. High‑level design (what we are building)

| Concern | How we satisfy it | Where the code lives |
|---------|-------------------|----------------------|
| **Configuration** | A single `retry_config.yaml` drives *all* thresholds (retry count, back‑off schedule, circuit‑breaker failure count, open timeout, fallback strategy). | `agent‑shield/config/retry_config.yaml` |
| **Retry** | Simple exponential back‑off that only retries *transient* errors (HTTP 5xx or `TimeoutError`). | Private helper `_retry_call` in `circuit_breaker.py` |
| **Circuit‑breaker** | Thread‑safe state machine (CLOSED → OPEN → HALF‑OPEN → CLOSED) with configurable failure threshold and open‑time. | `CircuitBreaker` class in `circuit_breaker.py` |
| **Fallback** | When the breaker is OPEN we return either a cached value (if the wrapped function supplies one) or a deterministic mock. | `_fallback` method – extensible via the `fallback` config key |
| **Logging** | Every state transition, retry attempt, and fallback hit is logged at `INFO`/`DEBUG` level with timestamps. | Throughout the module |
| **Public API** | One function `call_with_resilience(func, *args, **kwargs)` that composes *retry → circuit‑breaker → fallback* and can be imported everywhere. | Exported from `agent‑shield/src/retry/__init__.py` |
| **Testability** | The class is pure Python, uses only `time.sleep` (which can be monkey‑patched), and exposes its state via read‑only properties. | Unit‑tests can instantiate `CircuitBreaker()` directly. |

**1. Acceptance criteria**

- **Retry behavior** – When a tool call fails with a transient error (HTTP 5xx or a timeout), the system retries the call up to **3 times** with exponential back‑off delays of **1 s, 2 s, and 4 s** before giving up.
- **Circuit breaker activation** – After **5 consecutive failures** the circuit breaker transitions to the **open** state and blocks further calls for **30 s**.
- **Fallback activation** – While the circuit breaker is open, any attempted call immediately returns a **cached response** (or a predefined mock) without invoking the external service.
- **Logging** – Every retry attempt, circuit breaker state transition, and fallback activation is logged with a precise timestamp and relevant metadata (error code, attempt number, etc.).

**2. Unit tests** (pseudo‑code, Jest style)

```js
describe('CircuitBreaker', () => {
  let cb, mockToolCall, clock;

  beforeEach(() => {
    jest.useFakeTimers();
    clock = jest.spyOn(global, 'setTimeout');
    cb = new CircuitBreaker({maxFailures: 5, resetTimeout: 30000});
    mockToolCall = jest.fn();
  });

  afterEach(() => {
    jest.useRealTimers();
    clock.mockRestore();
  });

  test('retries 3 times on transient 5xx errors with exponential backoff', async () => {
    mockToolCall
      .mockRejectedValueOnce({status: 502})   // first fail
      .mockRejectedValueOnce({status: 503})   // second fail
      .mockResolvedValueOnce({data: 'ok'});   // third succeed

    const result = await cb.call(mockToolCall);

    expect(mockToolCall).toHaveBeenCalledTimes(3);
    expect(result.data).toBe('ok');
    // Verify backoff delays: 1s, 2s
    expect(clock).toHaveBeenNthCalledWith(1, expect.any(Function), 1000);
    expect(clock).toHaveBeenNthCalledWith(2, expect.any(Function), 2000);
  });

  test('fails after 3 retries', async () => {
    mockToolCall.mockRejectedValue({status: 504});
    await expect(cb.call(mockToolCall)).rejects.toMatchObject({status: 504});
    expect(mockToolCall).toHaveBeenCalledTimes(4); // initial + 3 retries
  });

  test('opens circuit after 5 consecutive failures', async () => {
    mockToolCall.mockRejectedValue({status: 500});
    for (let i = 0; i < 5; i++) {
      await expect(cb.call(mockToolCall)).rejects.toMatchObject({status: 500});
    }
    expect(cb.state).toBe('open');
  });

  test('blocks calls while circuit is open', async () => {
    cb.state = 'open';
    await expect(cb.call(mockToolCall)).rejects.toThrow('Circuit is open');
    expect(mockToolCall).not.toHaveBeenCalled();
  });

  test('closes circuit after reset timeout', async () => {
    cb.state = 'open';
    jest.advanceTimersByTime(30000);
    await cb.call(mockToolCall); // should attempt again
    expect(cb.state).toBe('half-open');
  });

  test('fallback returns cached data when circuit is open', async () => {
    cb.state = 'open';
    cb.fallback = jest.fn().mockReturnValue('cached');
    const res = await cb.call(mockToolCall);
    expect(res).toBe('cached');
    expect(cb.fallback).toHaveBeenCalled();
  });
});
```
