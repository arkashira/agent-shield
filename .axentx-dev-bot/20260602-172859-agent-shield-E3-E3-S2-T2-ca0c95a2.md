4. **Query endpoint** – `GET /audit/logs` supports query parameters `start_date`, `end_date`, `user_id`; returns matching records ordered by `timestamp` descending, with pagination (`limit`, `offset`).  
5. **Performance** – Logging adds **≤ 5 ms** latency to a memory operation under a sustained load of **10 k ops/sec** (measured with a synthetic benchmark).  
6. **Security & tamper‑evidence** – Each entry is signed with a server‑side HMAC; verification failures cause the request to be rejected and an alert logged.  
7. **Audit log accessibility** – Only users with the `audit_viewer` role can query logs; other authenticated users receive HTTP 403.
