+## Overview
+**agent‑shield** is a lightweight, extensible framework that lets you build, run, and secure AI agents in a SaaS‑centric environment.  
+It focuses on three core concerns:
+* **Tool‑call orchestration** – a declarative DSL for describing external tool integrations and a secure, rate‑limited gateway that routes calls.  
+* **Memory management** – a pluggable persistence layer that keeps agent state consistent across restarts and automatically prunes stale entries from a vector store.  
+* **Security hardening** – runtime checks that validate inputs, sanitize outputs, and log every request/response pair for auditability.
+
+## Quick‑Start
+```bash
+# Clone the repo
+git clone https://github.com/axentx/agent-shield.git
+cd agent-shield
+
+# Install dependencies
+pip install -r requirements.txt
+
+# Run the demo server
+python -m agent_shield.server
+
+# Test the endpoint
+curl -X POST http://localhost:8000/agent/run \
+  -H "Content-Type: application/json" \
+  -d '{"prompt":"Hello, world!"}'
+```
+
+```python
+from agent_shield import Agent, Tool
+
+class EchoTool(Tool):
+    def run(self, text: str) -> str:
+        return f"Echo: {text}"
+
+agent = Agent(name="EchoAgent", tools=[EchoTool()])
+print(agent.run("Hello, world!"))   # → Echo: Hello, world!
+```
+
+## Features
+- **Tool‑call routing** – automatically selects the best tool for a given intent.  
+- **Context pruning** – keeps the vector store size bounded while retaining high‑value embeddings.  
+- **Audit trail** – every request/response pair is logged with timestamps and metadata.  
+- **Extensible** – plug‑in new tools or memory back‑ends via the `agent_shield.plugins` package.  
+- **SaaS‑ready** – exposes a lightweight HTTP API that can be consumed by any SaaS‑based AI agent framework.
