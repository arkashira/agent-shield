 # agent-shield
 axentx product · An AI agent optimization platform that streamlines tool calls, memory management, and security for SaaS-based AI agents.
 
+--- 
+## Zero‑Config Deployment
+
+AgentOS Shield can be enabled **without touching any source code** – simply drop a YAML file into the AgentOS configuration directory and the runtime will pick it up automatically.
+
+### 1. Locate the AgentOS config directory
+The directory is resolved in the following order:
+1. The environment variable `AGENTOS_CONFIG_DIR` (e.g. `export AGENTOS_CONFIG_DIR=/etc/agentos`).
+2. If the variable is not set, the default location `./config` relative to the AgentOS executable is used.
+
+### 2. Create `agent‑shield.yaml`
+Create a file named **`agent-shield.yaml`** in the directory identified above.  
+Only the top‑level key `shield` is required; all sub‑keys are optional and have sensible defaults.
+
+```yaml
+# agent-shield.yaml – minimal configuration to enable the reliability layer
+shield:
+  enabled: true          # turn the Shield on
+  # optional: fine‑tune policies here
+  # policy:
+  #   timeout_seconds: 30
+```
+
+### 3. Start / restart AgentOS
+When the AgentOS process starts, it scans the config directory for `agent‑shield.yaml`.  
+If the file is present and valid, the Shield is activated automatically.
+
+### 4. Verify activation
+The system logs a clear confirmation message. Look for a line similar to:
+
+```
+INFO  [agent-shield] Shield activation successful – loaded configuration from /path/to/agent-shield.yaml
+```
+
+If the log line appears, the zero‑config deployment succeeded. No changes to the AgentOS source tree are required.
+
+### 5. No code changes required
+All you need is the YAML file; the existing AgentOS bootstrap code already contains the loader that searches for `agent‑shield.yaml`. This README update merely documents the process.
+
+--- 
+
 ## Usage
```
