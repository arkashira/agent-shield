import logging
from axentx.agent_shield import config

class RoleManager:
    def __init__(self):
        self.roles = {}

    def add_role(self, role):
        self.roles[role] = []

    def remove_role(self, role):
        if role in self.roles:
            del self.roles[role]

    def has_role(self, user_id, role):
        if role in self.roles:
            return user_id in self.roles[role]
        return False

    def revoke_role(self, role):
        if role in self.roles:
            self.roles[role].clear()

    def get_audit_logs(self):
        logs = []
        for role, users in self.roles.items():
            for user in users:
                logs.append(f"User {user} has role {role}")
        return logs

def get_role_manager():
    return RoleManager()

from typing import Dict

class RoleManager:

## 1. Acceptance Criteria
1. **CRUD operations** – The CLI can **create**, **list**, **update** (rename), and **delete** roles; the resulting role set persists across invocations.
2. **User‑role assignment** – The CLI can **assign** a role to a user and **revoke** a role from a user; a user may hold multiple roles.
3. **Enforcement** – When a tool call is invoked, the system checks the caller’s roles; the call succeeds **only** if the user possesses at least one role that matches the tool’s required role tag.
4. **Audit logging** – Every access decision (allow/deny) is recorded with **user_id**, **tool_name**, **required_role**, **decision**, and **ISO‑8601 timestamp**.
5. **Admin capabilities** – An admin user can:
   - Revoke any role from any user.
   - Query the audit log with optional filters (user_id, tool_name, date range).
6. **Error handling** – Invalid inputs (empty role name, unknown user, non‑existent role, malformed CLI flags) produce a non‑zero exit code and a clear error message; the program never crashes.
7. **Security** – Role checks are performed **before** any tool execution; logs are written to a file with permissions `600` (owner‑read/write only).
